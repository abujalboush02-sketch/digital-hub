OPERATIONAL BLUEPRINT

The Micro-SaaS Due Diligence Master Checklist

PURPOSE: Use this checklist before transferring any funds. This prevents you from buying a dying asset, a legal liability, or a technical nightmare.
01
Traffic & User Verification

Data Integrity

  • Request View-Only Analytics: Ask for Google Analytics, Plausible, or Fathom access. Do not accept screenshots.
  • Verify Active Users: Cross-reference database user counts with analytics active user data.

Traffic Quality

  • Bot Detection: Check "Average Session Duration" and "Bounce Rate". If average time on site is < 5 seconds, it is likely bot traffic.
  • Fragility Check: Ensure traffic isn't reliant on a single source. Look for consistent organic/direct traffic.
02
Financial Audit

Revenue Verification

  • Direct Access: Request view-only access to Stripe or Lemon Squeezy.
  • MRR vs. Spikes: Ensure Monthly Recurring Revenue is actually recurring and not just a spike from lifetime deals.

The Profit Margin

  • Churn Rate: If they add 10 users but lose 9, the core product is broken.
  • True Expense Audit: Subtract all costs (AWS, Vercel, Mailgun, API credits) from MRR to find true profit.
Operator's Warning: The Analytics Trap

Never base an acquisition on a shared screen or a PDF report. High-level scammers can easily spoof dashboard UI using browser developer tools. View-only access to the source data is the only non-negotiable proof of health.

03
Technology & Codebase

Technical Health

  • Stack Review: Ensure standard frameworks (React, Node, Python, etc.). Reject obscure or obsolete languages.
  • Dependency Audit: Check if core packages are dangerously out of date.

Infrastructure

  • Hosting Complexity: Can it run on Vercel/Heroku, or does it require expensive, complex AWS architecture?
  • Database Handoff: Confirm exact transfer method (e.g., Postgres dump or MongoDB transfer).
04
Payment Gateway & Legal

Compliance Check

  • MoR Transition: If you are in an unsupported Stripe country, ensure the API allows a quick swap to Lemon Squeezy.
  • TOS Alignment: Ensure the app doesn't violate 3rd party rules.

Ownership Rights

  • IP Ownership: Get written confirmation that the founder owns 100% of the code and is transferring all IP rights.
05
The Handoff (Escrow Protocol)

Execution Steps:

  • Set up an Escrow.com or similar secure transaction.
  • Transfer Domain Name via registrar push.
  • Transfer GitHub/GitLab repository ownership.
  • Transfer all Social Media accounts (X, LinkedIn, etc.).
  • RELEASE FUNDS ONLY after compiling the code locally and verifying it runs.

AUTHORIZATION REQUIRED

This document serves as the primary filter for all Micro-SaaS acquisitions. Any asset that fails more than two checks in Section 01 or 02 should be discarded immediately.
